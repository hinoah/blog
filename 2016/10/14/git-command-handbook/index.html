git常用命令速查手册

2016-10-14 • 分类: 技术 • 标签: git

接触git也有一年多的时间了，但对于命令行模式还不是特别熟悉。想来也是，之前在eclipse上用e-git插件，现在转idea后，其提供的git插件依旧强大。最近开始写博客并托管在github上，再把代码搬到ide上就显得有点重了，所以开始使用git命令，但本人忘性太大，一忘就得上网查，但网上大多教程格式凌乱，所有命令堆叠在一起，没什么重点，阅读起来感觉很不友好。所以我针对自己的需求，在这里整理一些常用的git命令，便于速查。

常用命令

git add

用于跟踪新增和改动的文件，如果修改后的文件不进行git add，则不会被提交到本地仓库。
比较常用的：

  • git add . (添加所有改动的文件)
  • git add <path> (添加某个路劲下的改动文件或单独指定某个文件)
  • git add -i (查看当前改动过文件的提交情况)

git commit

提交命令，先介绍一下本地仓库文件可能处在的三种状态。

  1. Untracked files 文件未被跟踪
  2. Changes to be committed 文件已被提交
  3. Changes not staged for commit 文件被修改，但还未被修改

这三种状态下的文件列表可以用git status查看

在第一种和第三种状态下的文件可以使用git add命令, 提交到本地仓库中（使其状态变为第二种Changes to be committed）
